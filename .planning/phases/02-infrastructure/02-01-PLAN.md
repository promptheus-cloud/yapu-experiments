    **2. Create lib/deep-merge.ts:**
    A simple recursive deep-merge utility for the patch operation. Uses nested object format (not dot-notation) per research decision.

    ```typescript
    export function deepMerge<T extends Record<string, unknown>>(
      target: T,
      source: Record<string, unknown>
    ): T {
      const output = { ...target };
      for (const key of Object.keys(source)) {
        const sourceVal = source[key];
        const targetVal = (target as Record<string, unknown>)[key];
        if (
          sourceVal &&
          typeof sourceVal === 'object' &&
          !Array.isArray(sourceVal) &&
          targetVal &&
          typeof targetVal === 'object' &&
          !Array.isArray(targetVal)
        ) {
          (output as Record<string, unknown>)[key] = deepMerge(
            targetVal as Record<string, unknown>,
            sourceVal as Record<string, unknown>
          );
        } else {
          (output as Record<string, unknown>)[key] = sourceVal;
        }
      }
      return output;
    }
    ```

    Arrays are replaced entirely (not merged element-by-element) — this is intentional for content editing where replacing an array of sections is the expected behavior.

    **3. Create app/api/content/route.ts:**
    A POST route handler implementing EDIT-01. Must include:

    a. **Auth check:** Read `Authorization` header, compare to `Bearer ${process.env.CONTENT_API_SECRET}`. Return 401 if mismatch.

    b. **Body validation with Zod:**
    ```typescript
    const BodySchema = z.object({
      locale: z.enum(['en', 'fr', 'es']),
      file: z.string().regex(/^[a-z][a-z0-9-]*$/, 'File name must be lowercase alphanumeric with hyphens only'),
      content: z.record(z.unknown()).optional(),
      patch: z.record(z.unknown()).optional(),
    }).refine(
      (d) => Boolean(d.content) !== Boolean(d.patch),
      { message: 'Provide exactly one of content (full replace) or patch (partial merge)' }
    );
    ```
    - `locale` restricted to the three supported locales
    - `file` restricted to `^[a-z][a-z0-9-]*$` — prevents path traversal (no slashes, no dots, must start with letter)
    - Must provide exactly ONE of `content` (full replace) or `patch` (partial merge)

    c. **File existence check:** Before writing, verify the target file exists (prevents creating arbitrary new files via the API). If file does not exist, return 404 `{ error: 'Content file not found' }`.

    d. **Write or patch:**
    - If `content` provided: write the entire object as JSON with 2-space indentation
    - If `patch` provided: read existing file, deep-merge with patch, write result
    - Use `fs.promises.writeFile` (async, not sync) and `fs.promises.readFile`

    e. **Response:** Return `{ ok: true, locale, file }` on success.

    f. **Error handling:** Wrap the file operations in try/catch. Return 500 with `{ error: 'Failed to write content' }` on filesystem errors.

    g. **CORS headers:** The route handler should handle OPTIONS preflight requests and include CORS headers (Access-Control-Allow-Origin: *, Access-Control-Allow-Headers: Authorization, Content-Type, Access-Control-Allow-Methods: POST, OPTIONS) for cross-origin Ember access.

    h. **Do NOT add `export const dynamic = 'force-dynamic'` to the API route** — Route Handlers that use Request object (like reading headers) are automatically dynamic in Next.js App Router.
