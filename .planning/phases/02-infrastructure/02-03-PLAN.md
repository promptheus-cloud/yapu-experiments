<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Deploy to VPS and verify infrastructure</name>
  <action>
    The user must perform the following steps on the Promptheus VPS. All config files were created in Plans 01 and 02.
  </action>
  <how-to-verify>
    **Pre-deployment: DNS Setup**
    1. Create two A records in your DNS provider:
       - `yapu.promptheus.cloud` -> VPS IP address
       - `yapu-preview.promptheus.cloud` -> VPS IP address
    2. Wait for DNS propagation (5-30 minutes). Verify with: `dig yapu.promptheus.cloud +short`

    **VPS Deployment Steps**
    3. SSH to the VPS: `ssh root@<VPS_IP_ADDRESS>`
    4. Verify available ports: `ss -tlnp | grep -E '300[0-9]'` — confirm 3002 and 3003 are free. If not, update `server/ecosystem.config.js` and both nginx configs with available ports.
    5. Verify existing tools: `nginx -v && certbot --version && pm2 --version && node --version`
    6. Clone or update the repo:
       ```bash
       mkdir -p /home/yapu && cd /home/yapu
       git clone <repo-url> yapu2 || (cd yapu2 && git pull origin master)
       cd /home/yapu/yapu2
       ```
    7. Install dependencies: `npm ci`
    8. Create environment file:
       ```bash
       cp .env.example .env.local
       # Edit and set real tokens:
       nano .env.local
       # Generate tokens: openssl rand -base64 32
       ```
    9. Copy nginx configs:
       ```bash
       cp server/nginx-yapu-prod.conf /etc/nginx/sites-available/yapu-prod
       cp server/nginx-yapu-preview.conf /etc/nginx/sites-available/yapu-preview
       ln -sf /etc/nginx/sites-available/yapu-prod /etc/nginx/sites-enabled/
       ln -sf /etc/nginx/sites-available/yapu-preview /etc/nginx/sites-enabled/
       ```
    10. Edit preview nginx config — replace `PREVIEW_ACCESS_TOKEN_PLACEHOLDER` with the real PREVIEW_ACCESS_TOKEN from .env.local:
        ```bash
        nano /etc/nginx/sites-available/yapu-preview
        ```
    11. Test nginx config: `nginx -t`
    12. Reload nginx: `systemctl reload nginx`
    13. Obtain SSL certificates:
        ```bash
        certbot --nginx -d yapu.promptheus.cloud
        certbot --nginx -d yapu-preview.promptheus.cloud
        ```
    14. Build production: `npm run build` (writes to .next-prod/)
    15. Start PM2 processes:
        ```bash
        pm2 start server/ecosystem.config.js
        pm2 save
        pm2 startup  # follow instructions to enable reboot persistence
        ```

    **Verification Checks**
    16. Production site accessible: `curl -sI https://yapu.promptheus.cloud | head -5` — expect `HTTP/2 200`
    17. Preview without auth blocked: `curl -sI https://yapu-preview.promptheus.cloud | head -5` — expect `HTTP/1.1 403`
    18. Preview with auth accessible: `curl -sI -H "Authorization: Bearer YOUR_PREVIEW_TOKEN" https://yapu-preview.promptheus.cloud | head -5` — expect `HTTP/2 200`
    19. Content API auth check: `curl -s -X POST https://yapu.promptheus.cloud/api/content -H "Content-Type: application/json" -d '{"locale":"en","file":"homepage","patch":{"hero":{"title":"Test Edit"}}}' | head -5` — expect `{"error":"Unauthorized"}`
    20. Content API write test:
        ```bash
        curl -s -X POST https://yapu.promptheus.cloud/api/content \
          -H "Authorization: Bearer YOUR_CONTENT_API_SECRET" \
          -H "Content-Type: application/json" \
          -d '{"locale":"en","file":"homepage","patch":{"hero":{"title":"YAPU Test Edit"}}}'
        ```
        Expect: `{"ok":true,"locale":"en","file":"homepage"}`
    21. Verify content change visible: `curl -s https://yapu.promptheus.cloud/en/ | grep "YAPU Test Edit"` — should find the updated title
    22. Revert test edit:
        ```bash
        curl -s -X POST https://yapu.promptheus.cloud/api/content \
          -H "Authorization: Bearer YOUR_CONTENT_API_SECRET" \
          -H "Content-Type: application/json" \
          -d '{"locale":"en","file":"homepage","patch":{"hero":{"title":"Welcome to YAPU Solutions"}}}'
        ```
    23. PM2 reboot test: `pm2 list` — both yapu-prod and yapu-dev should show status "online"
  </how-to-verify>
