    **PartnerCarousel.tsx** — Server component (NO `"use client"`). Receives logos array and section title as props.

    ```typescript
    interface Logo {
      alt: string;
      src: string;
    }

    interface PartnerCarouselProps {
      title: string;
      logos: Logo[];
    }
    ```

    Renders a `<section>` with `py-12 px-6 overflow-hidden` and `aria-label={title}`. Inside: `max-w-6xl mx-auto`. Section title H2 with `text-2xl font-bold text-center mb-8 text-foreground`. Then the carousel wrapper:

    ```tsx
    <div className="overflow-hidden group">
      <div className="flex w-max animate-scroll-logos group-hover:[animation-play-state:paused]">
        {[...logos, ...logos].map((logo, i) => (
          <div key={i} className="mx-8 flex-shrink-0 flex items-center justify-center">
            <div className="h-10 w-24 bg-muted rounded flex items-center justify-center text-xs text-muted-foreground truncate px-1">
              {logo.alt}
            </div>
          </div>
        ))}
      </div>
    </div>
    ```

    **Critical:** The logo list is rendered TWICE (`[...logos, ...logos]`) to create the seamless infinite loop. The `translateX(-50%)` in the CSS animation scrolls through the first copy and loops back. Phase 3 uses gray placeholder `<div>` boxes (not `<img>`) because real logo images are a Phase 4 concern. The placeholder shows the logo alt text for identification.
