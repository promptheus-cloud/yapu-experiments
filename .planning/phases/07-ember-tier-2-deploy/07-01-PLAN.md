
Using `PM2_PROD_APP` env var makes this configurable per customer. Default is `yapu-prod` for YAPU. Other Ember instances can set their own PM2 app name. If the env var is not set, `pm2 restart yapu-prod` works because PM2 can restart by process name (no ecosystem file path needed).

In `C:/Users/hmk/promptheus/yapu2/server/ecosystem.config.js`, add `exec_mode: "fork"` to BOTH PM2 entries. This is critical for `yapu-dev` (cluster_mode breaks `next dev` HMR) and harmless for `yapu-prod` (1 instance in fork mode is equivalent):
