## Summary

Phase 04.2 closes the remaining visual gaps between the local rebuild and the original yapu.solutions homepage and subpages. Phase 04.1 fixed CSS-level issues (colors, button shapes, backgrounds, typography). Phase 04.2 must address **structural and content-level** differences that remain.

The ground-truth evidence is the side-by-side screenshot comparison in `screenshots/home-desktop-original.png` vs `screenshots/home-desktop-local.png`. These screenshots reveal five categories of remaining gaps: (1) hero headline and CTA text do not match the original, (2) the service modules section uses a 4-column icon card grid instead of the original's 4 large full-width alternating sections, (3) the Newsletter section has the wrong background color (light mint vs dark teal), (4) the hero content text differs from yapu.solutions, and (5) the Testimonial section title and structure is slightly off.

The biggest gap — and the one requiring most work — is the **ServiceModules restructuring**: the original has 4 separate full-width sections stacked vertically with bullet point lists and "read more" links, while the local has a compact 4-column card grid. This is a layout/component rebuild, not just a CSS tweak.

**Primary recommendation:** Fix in priority order: (1) hero text content, (2) ServiceModules full-width section layout, (3) Newsletter background, (4) Testimonial title text. Use the Puppeteer visual-compare script to measure diff % before/after each change.

### Pattern 1: ServiceModules — Stacked Full-Width Sections

**What:** The original yapu.solutions homepage shows 4 large full-width stacked sections below the hero, NOT a 4-column card grid. Each section:
- Takes the full page width
- Has a distinctive background color (alternating: light gray `#f8f8f8`, off-white `#f8f8f8`, off-white, dark teal `#1E5A64`)
- Contains the service title (bold, ~2em), a bullet list of 3-4 features, and a "read more" link
- Does NOT use shadcn Card component — it is a plain section layout
- Layout: text content block approximately 50-60% wide, left or center-aligned
- Section height: substantial (200-300px with 4em padding)

**Screenshot evidence:** In `home-desktop-original.png`, the 4 service sections are clearly visible as distinct full-width bands, each with colored backgrounds: Investor Services (light gray), Data Insights (off-white), Digital Tools (off-white), Impact (dark teal with white text).

**From WebFetch analysis of yapu.solutions:**
- Investor Services: background `rgb(217,217,217)` (slightly darker gray), bullet points: Path to resilience, Measure your impact, Monitor your portfolio in real time
- Data Insights: background `rgb(248,248,248)` (off-white), bullet points: Loan decision support, Climate/environmental/social risks, Biodiversity risk, Staff performance
- Digital Tools: background `rgb(248,248,248)` (off-white), bullet points: Easy digital transformation, Improved risk management, Data insights
- Impact: background `rgb(30,90,100)` (dark teal = `bg-brand`), white text, bullet points: Putting financial inclusion first, Promoting green finance, Fostering climate resilience for the most vulnerable

**Each section has a "read more" link** pointing to the respective subpage.

**Correct implementation pattern:**

```tsx
// ServiceModules.tsx — replace 4-column card grid with stacked sections
const sectionBgs = {
  'investor-services': 'bg-[rgb(217,217,217)]',
  'data-insights': 'bg-gray-50',
  'digital-tools': 'bg-gray-50',
  'impact': 'bg-brand text-white',
};

// Each module in the new ServiceModule interface needs a "bullets" array
// and a "readMoreText" string
interface ServiceModule {
  id: string;
  title: string;
  bullets: string[];
  readMoreText: string;
  href: string;
}

// Render as stacked sections:
export function ServiceModules({ modules }: ServiceModulesProps) {
  return (
    <>
      {modules.map((module) => {
        const isImpact = module.id === 'impact';
        return (
          <section
            key={module.id}
            className={`py-16 px-6 ${isImpact ? 'bg-brand text-white' : 'bg-[#f8f8f8]'}`}
          >
            <div className="max-w-4xl mx-auto">
              <h2 className={`text-2xl font-bold mb-6 ${isImpact ? 'text-white' : 'text-brand'}`}>
                {module.title}
              </h2>
              <ul className="space-y-2 mb-6">
                {module.bullets.map((bullet, i) => (
                  <li key={i} className="flex items-center gap-2">
                    <span className="w-1.5 h-1.5 rounded-full bg-current flex-shrink-0" />
                    <span>{bullet}</span>
                  </li>
                ))}
              </ul>
              <Link href={module.href} className={`text-sm font-semibold ${isImpact ? 'text-accent' : 'text-cta'} hover:underline`}>
                {module.readMoreText}
              </Link>
            </div>
          </section>
        );
      })}
    </>
  );
}
```

**Note on homepage.json content update:** The `serviceModules` array needs to be updated with `bullets` array instead of `description` string, and add `readMoreText`. The existing `icon` field can be dropped. Update all 3 locales (en/fr/es).

## Common Pitfalls

### Pitfall 1: Breaking the ServiceModules JSON schema

**What goes wrong:** Adding `bullets` array to `serviceModules` items breaks TypeScript interface in `page.tsx` (currently typed with `description: string` and `icon: string`).

**Why it happens:** The content JSON schema is typed inline in `app/[locale]/page.tsx` as `ServiceModule` interface.

**How to avoid:** Update the `ServiceModule` interface in `page.tsx` when updating the JSON. Remove `icon` and `description` fields, add `bullets: string[]` and `readMoreText: string`. The `ServiceModules` component also needs its prop interface updated.

**Warning signs:** TypeScript error `Property 'bullets' does not exist on type 'ServiceModule'`.

### Pitfall 2: Newsletter form contrast on dark background

**What goes wrong:** Changing Newsletter background to `bg-brand` (dark teal) makes input placeholders and labels invisible (they're dark text on dark background by default).

**Why it happens:** shadcn Input component uses `--background` color which is white/light.

**How to avoid:** Override input styling on dark newsletter section: inputs need `bg-white/10 text-white placeholder:text-white/60 border-white/30`. Or use `bg-white text-brand` for inputs (white input on dark bg is more readable).

**Warning signs:** Input fields invisible or placeholder text unreadable.

### Pitfall 3: Locale JSON update lag

**What goes wrong:** Updating `content/data/en/homepage.json` but forgetting `fr/` and `es/` files, causing broken FR/ES pages.

**Why it happens:** Content lives in 3 separate JSON files per locale.

**How to avoid:** Always update all 3 locale files (en/fr/es) in the same task. The `readContent` function falls back to EN if the locale file is missing, but the schema change (removing `description`/`icon`, adding `bullets`/`readMoreText`) would break if EN and FR/ES have different shapes.

**Warning signs:** TypeScript error on FR/ES page at build time, or FR/ES pages showing EN fallback unexpectedly.

### Pitfall 4: ServiceModules "title" prop removal

**What goes wrong:** The current `ServiceModules` component accepts a `title` prop (used for "Our Services" section heading). The original does NOT have a heading above the 4 sections — the sections themselves are the headings. Removing the outer `title` prop means updating the call in `page.tsx`.

**Why it happens:** The current design added a section title ("Our Services") that doesn't exist in the original.

**How to avoid:** In the restructured `ServiceModules`, remove the `title` prop entirely. In `page.tsx`, remove the `servicesTitle` i18n call for this section.

**Warning signs:** "Our Services" heading still visible above the 4 service sections.

### Pitfall 5: Investor Services background color precision

**What goes wrong:** The Investor Services section uses `rgb(217,217,217)` (a noticeably darker gray) while Data Insights and Digital Tools use `rgb(248,248,248)` (near-white). Using `bg-gray-50` (`#f9fafb`) for all three makes Investor Services look lighter than the original.

**How to avoid:** Use `bg-[rgb(217,217,217)]` for Investor Services section, `bg-gray-50` for Data Insights and Digital Tools.

**Warning signs:** All three non-Impact sections look the same shade, instead of Investor Services being slightly darker.

### ServiceModules: Complete Restructured Component

```tsx
// components/ServiceModules.tsx — stacked full-width sections
import { Link } from '@/i18n/navigation';

interface ServiceModule {
  id: string;
  title: string;
  bullets: string[];
  readMoreText: string;
  href: string;
}

interface ServiceModulesProps {
  modules: ServiceModule[];
}

const sectionConfig: Record<string, { bg: string; titleColor: string; textColor: string; linkColor: string }> = {
  'investor-services': {
    bg: 'bg-[rgb(217,217,217)]',
    titleColor: 'text-brand',
    textColor: 'text-foreground',
    linkColor: 'text-cta',
  },
  'data-insights': {
    bg: 'bg-gray-50',
    titleColor: 'text-brand',
    textColor: 'text-foreground',
    linkColor: 'text-cta',
  },
  'digital-tools': {
    bg: 'bg-gray-50',
    titleColor: 'text-brand',
    textColor: 'text-foreground',
    linkColor: 'text-cta',
  },
  'impact': {
    bg: 'bg-brand',
    titleColor: 'text-white',
    textColor: 'text-white',
    linkColor: 'text-accent',
  },
};

export function ServiceModules({ modules }: ServiceModulesProps) {
  return (
    <>
      {modules.map((module) => {
        const config = sectionConfig[module.id] ?? sectionConfig['data-insights'];
        return (
          <section key={module.id} className={`${config.bg} py-16 px-6`}>
            <div className="max-w-4xl mx-auto">
              <h2 className={`text-2xl font-bold mb-6 uppercase tracking-wide ${config.titleColor}`}>
                {module.title}
              </h2>
              <ul className={`space-y-3 mb-8 ${config.textColor}`}>
                {module.bullets.map((bullet, i) => (
                  <li key={i} className="flex items-center gap-3 text-sm md:text-base">
                    <span className="w-1.5 h-1.5 rounded-full bg-current flex-shrink-0 opacity-70" />
                    {bullet}
                  </li>
                ))}
              </ul>
              <Link
                href={module.href}
                className={`text-sm font-semibold ${config.linkColor} hover:underline uppercase tracking-wider`}
              >
                {module.readMoreText}
              </Link>
            </div>
          </section>
        );
      })}
    </>
  );
}
```
